遇到的崩溃

Unrecognized Selector

有一次误删了项目中崩溃防护的库，导致版本更新后 Organizer 中显示大量来自第三方库而且以前从没出现过的崩溃，由此知道原来有些崩溃是可以通过 Runtime 做全局防护的。

项目中的例子：protocol 是 @optional，所以我们本地代码并没有提供实现，但 SDK 内部调用时没有判断 delegate respondsToSelector。

解决：提供空实现

容器类插入 nil

解决：插入时使用宏校验参数

除以 0

定位楼层，按比例在地图上绘制大楼，服务器传的总楼层数为 0，导致除以 0。

非主线程刷新 UI

NSNotification

iOS 9 之前，在 NSNotificationCenter 添加一个对象为观察者后，没有在对象 dealloc 时移除观察者。

KVO

  1. KVO 添加和移除次数不匹配：移除未注册的观察者，移除多次
  2. 被观察者在 dealloc 时仍注册者 KVO
  3. 添加了观察者，未实现 observeValueForKeyPath:ofObject:change:context: 方法
  4. keyPath == nil
