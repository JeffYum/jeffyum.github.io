接入 CallKit 的大致框架。

VoIP

VoIP push，提供区别于普通 APNS push 的能力，通过这种 push 方式收到消息时会直接将未启动的 app 激活。也就是说，消息推过来后，能够直接运行回调函数里的代码。iOS 13 之后，VoIP 必须配合 CallKit 使用，否则系统会停止启动 app。PushKit 和 APNS 推送是共存的，所以在系统里需要两个证书。

配置

  1. Background Modes 勾选 Voice over IP

  2. 创建 PushKit 推送证书，证书创建类型选择 VoIP Services Certificate

  3. Build Phases -> Link Binary With Libraries -> 添加 PushKit

App 与系统通信

CXProviderDelegate

由系统触发的回调方法，CXProviderDelegate 定义了 provider 生命周期事件（带有 CXAction 子类参数），原本 app 内通话相关的方法应该在这里调用。

CXAction

CXAction 是代表电话操作的抽象类，CallKit 为不同 action 提供不同的具体实现，例如，CXStartCallAction 代表初始化去电，CXAnswerCallAction 代表接听电话。每一个 action 都有唯一的 UUID。

CXCallController

App 使用 CXCallController 将用户发起的请求通知给系统，例如开始/结束/静音/挂起通话动作，也就是说，这些动作都是在 app 内触发（如下图），而非系统的通话界面。call controller 使用 CXTransaction 发起请求，CXTransaction 包含一个或多个 CXAction 实例，call controller 向系统发送 transaction，如果没有错误发生，系统将触发相应的 CXProviderDelegate 回调。
