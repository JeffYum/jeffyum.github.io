Stack View

发表于 2018-10-23 更新于 2022-08-09 分类于 iOS Disqus：

There is still no code in this chapter. We learn how to add images in our app, we are recommand to use vector images and preserve them. Stack view is a powerful tool to minimize the number of constraints. Two or more objects are embed in one stack, so that we can easily manage them. Sometimes, we hope a layout just displays on a specific screen, such as iPhone landscape. We also fixed some issues, for example, some fonts maybe too big on small screen iPhone, button becomes narrower after embed.

We can group multiple objects together so that they can moved or resized as a single object.

Arrange view: views that are embedded in a stack view.

Add Images

Assets.xcassets: manage images and icons

When PNG is used, Apple recommends developers to provide three resolutions of images

  • @3x iPhone Plus and iPhone X
  • @2x SE/6/7/8
  • Without @ non-Retina display

Whenever possible, ask your designer to prepare PDF format, because it can be scalable without losing quality.

To retain the aspect ratio of image, in the document outline view, control-drag horizontally on the image view of user1, user2, user3

Two ways to use it

  1. Drag from the Object library, put it right into the storyboard. Then drag and drop other view objects into the stack view.
  2. Select two or more view objects and click Stack button in the auto layout bar.

Tips: shift + right-click can help you select objects

Properties of stack view

  • Axis: vertical, horizontal
  • Alignment: leading, center, trailing
  • Distribution: fill, fill equally

In the past, I always can’t distinguish fill and fill equally. Fill equally means every arranged views are the same size.

Issues Fixing

  1. The Instant Developer label is too long to display perfectly on iPhone SE or smaller machine.

​ Attibutes inspector -> Autoshrink -> Minimum Font Size -> 20

  1. Button is narrower than expected as we embed the buttons in the stack view.

​ Set a width constraint to the stack view. In document view, control-drag from the stack view to itself.
